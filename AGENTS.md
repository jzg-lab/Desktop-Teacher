# AGENTS.md

## Purpose

This file is a navigation map for future agents. Keep it short. Durable truth belongs in the docs it points to.

## Collaboration Stance

- Pressure-test assumptions.
- Prefer direct critique over reassurance.
- Treat vague plans as incomplete until they are executable.

## Authority Order

1. Current repository docs that define product and system behavior.
2. Execution-plan docs that define sequence and current implementation order.
3. Historical chat context only when the repository docs are silent.
