# 步骤 01：桌面壳、角色图标与侧边窗骨架

状态：Planned  
目标版本：V0 / MVP

## 目标

先把用户能看到、能唤起的桌面形态跑通，包括：

- 右下角角色图标
- 角色状态反馈
- 侧边回答窗容器

## 依赖

- [step-00-tech-foundation.md](/mnt/e/my_github/Desktop-Teacher/docs/exec-plans/step-00-tech-foundation.md)

## 任务

1. 实现应用启动后的常驻入口。
2. 实现右下角角色图标。
3. 定义并接通角色状态：
   - 空闲
   - 处理中
   - 错误
4. 实现侧边回答窗的空壳容器。
5. 打通图标与侧边窗的打开、关闭、收起。

## 产出

- 角色图标基础实现
- 侧边窗基础容器
- 状态联动框架

## 完成标准

- 启动应用后，用户可以看到角色图标
- 用户可以通过角色图标打开侧边窗
- 侧边窗支持关闭与收起

## 验证

- 手动验证图标显示位置
- 手动验证侧边窗打开 / 关闭 / 收起
- 重启应用后复测入口稳定性
